# Verrocchio — App Privacy Questionnaire (App Store Connect)

Founder ticks through the ASC App Privacy section using this exact sequence. Pasted from `docs/research/2026-05-12-appstore-readiness/01-privacy.md` Artifact 2 with ASC questionnaire phrasing.

## Step 1 — Data Collection

**"Do you or your third-party partners collect data from this app?"** → **Yes**

Tap **Set Up Data Types**.

## Step 2 — Data Types Collected

For each category below, tick "Yes — collected" and answer the follow-up questions. Categories NOT listed are NOT collected.

### Contact Info
- **Email Address** → Yes
  - Linked to user's identity? **Yes**
  - Used for tracking? **No**
  - Purpose: **App Functionality**

### Identifiers
- **User ID** → Yes  (Firebase Authentication UID)
  - Linked to user's identity? **Yes**
  - Used for tracking? **No**
  - Purpose: **App Functionality**

### User Content
- **Other User Content** → Yes  (journal entries, goals, todos, habits, quotes, archive, daily ritual notes, upcoming dates, custom categories)
  - Linked to user's identity? **Yes**
  - Used for tracking? **No**
  - Purpose: **App Functionality**

- **Photos or Videos** → Yes  (only if user uploads media to Firebase Storage from within the app)
  - Linked to user's identity? **Yes**
  - Used for tracking? **No**
  - Purpose: **App Functionality**

### Health & Fitness
- **Fitness** → Yes  (habit completions, streaks, durations/units like minutes, cups, pages — self-reported by the user)
  - Linked to user's identity? **Yes**
  - Used for tracking? **No**
  - Purpose: **App Functionality**

### Usage Data
- **Product Interaction** → Yes  (which dates the user opened the app, feature-access timestamps, onboarding/tour flags)
  - Linked to user's identity? **Yes**
  - Used for tracking? **No**
  - Purpose: **App Functionality**

### Other Data
- **Other Data** → Yes  (gamification state: XP, best streaks, unlocked achievements; preferences: sort order, AI tone, custom goal types, custom importance levels, custom sections; self-reported coarse home location and travel days; signup timestamp)
  - Linked to user's identity? **Yes**
  - Used for tracking? **No**
  - Purpose: **App Functionality**

## Categories NOT collected

Tick "No" for all of these in ASC's matrix:

- Financial Info (Payment Info, Credit Info, Other Financial Info) — none
- Location (Precise Location, Coarse Location) — coarse only as user-typed text; if ASC insists, declare "Coarse Location" under Other Data as above. Do NOT tick the GPS-based Location categories.
- Sensitive Info (Race/Ethnicity, Sexual Orientation, etc.) — none
- Contacts — none
- Browsing History — none
- Search History — none
- Diagnostics (Crash Data, Performance Data, Other Diagnostic Data) — none currently; if you add Crash SDK later, this changes
- Purchases — none
- Other Financial Info — none

## Step 3 — Tracking

**"Do you or your third-party partners use data to track users?"** → **No**

Verrocchio does NOT cross-app track, fingerprint, or use IDFA. No App Tracking Transparency prompt is required.

If this changes in a future version (e.g., adding Firebase Analytics or a crash SDK), ASC privacy responses MUST be updated AND the App Tracking Transparency prompt MUST be implemented before that version submits.

## Step 4 — Publish

Tap **Publish** on the App Privacy page. The Privacy Nutrition Label generated by ASC should display:

- **Data Used to Track You:** None
- **Data Linked to You:**
  - Contact Info → Email Address
  - Identifiers → User ID
  - User Content → Other User Content, Photos or Videos
  - Health & Fitness → Fitness
  - Usage Data → Product Interaction
  - Other Data → Other Data
- **Data Not Linked to You:** None

## Notes for any future change

If Verrocchio later adds:
- **Firebase Analytics or any third-party analytics SDK** → declare Diagnostics + likely change Tracking answer.
- **Crash reporting (Sentry, Bugsnag, Firebase Crashlytics)** → declare Diagnostics → Crash Data.
- **Advertising** → declare a tracking purpose; ATT prompt required.
- **AI sub-processor (Anthropic) actively used at launch** → declare User Content → Other User Content with purpose still App Functionality (per the policy: Anthropic does not train on user content).
