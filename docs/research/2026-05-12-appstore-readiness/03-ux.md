## 1. BLOCKERS (App Store 4.0 quality bar)

- **UX-B-01 — AI buttons crash into a server-error message in v1.0.** `AI_BACKEND_URL = null` means every "Generate insight / Generate briefing" tap shows a developer-facing fallback. Apple reviewers WILL tap these. Fix: hide AI-affordance buttons when `!AI_BACKEND_URL` (single feature flag at top of `index.html`). Guideline 2.1.
- **UX-B-02 — Demo personas + seeded password ship in v1.0.** `DEMO_PASSWORD = "verrocchio-demo-1"` and four `@demo.verrocchio.app` accounts are live Firebase users with hardcoded shared credential. Fix: gate `DEMO_PERSONAS` UI behind a non-production flag.
- **UX-B-03 — No password reset flow.** Login screen exposes login + signup only. Fix: add "Forgot password?" link → `auth.sendPasswordResetEmail(email)` with generic success message (anti-enumeration).
- **UX-B-04 — Auth error strings are raw Firebase codes.** Fix: map Firebase error codes to friendly copy.
- **UX-B-05 — Portrait-locked PWA with desktop CSS rules.** Inconsistent iPad behavior. Decision (with iOS-H-09): iPhone-only at launch, drop iPad support.
- **UX-B-06 — No Sign in with Apple option.** Not strictly required for v1.0; revisit v1.1.
- **UX-B-07 — `splashPhase` → login flash.** Hold splash until `authUser` resolves definitively.
- **UX-B-08 — Privacy policy + terms link missing from sign-up screen.** Add link below sign-up button.

## 5. Onboarding decision

**Recommendation: single-screen welcome modal.** Dismissible, one CTA "Add your first habit," one "Skip for now." Don't restore 3-screen carousel or full tour.

Implementation: on first signed-in render with `!data.habits?.length && !data.tourDone`, show modal; on dismiss or "add habit" tap, set `tourDone: true`.

## 6. AI feature "off" UX

**Plan: hide AI buttons entirely for v1.0** via `const AI_ENABLED = !!AI_BACKEND_URL;` feature flag. Re-enable in v1.1 once proxy is deployed and reviewed.

## 7. Account-deletion UX flow

1. Profile → "Danger zone" disclosure (collapsed).
2. "Delete account" — red text, not red-filled button.
3. Confirm dialog: "This permanently deletes your account, all habits, journal entries, goals, and uploaded files. This cannot be undone."
4. Type-to-confirm field: "delete my account".
5. Password re-auth field.
6. "I forgot my password" link triggers `sendPasswordResetEmail`.
7. On success: sign out, return to login with toast.

## 8. Safe-area-inset audit targets

- Sticky top header — top inset.
- Fixed bottom nav — bottom inset.
- Modal close buttons + bottom sheets — top inset.
- Keyboard-avoiding input rows (journal, habit add, todo add) — `KeyboardResize.body` plugin or `scrollIntoView` on focus.
- Floating undoToast — `bottom: max(120px, calc(env(safe-area-inset-bottom) + 80px))`.
- Auth screen bottom buttons — home indicator overlap.

**Audit method (plan task W3-T8):** grep inline `position: 'fixed'` and `position: 'sticky'` — every match needs review.

## 10. Touch target audit (HIG 44×44pt min)

Grep patterns (in plan task W4-T3):
- `onClick=` on `span` / text-only.
- `fontSize: '12'` adjacent to `onClick=`.
- Icon-only buttons missing `aria-label`: `>×<`, `>✕<`, `>⋯<`, `>+<`.
- `padding: '4px'` on interactive.
- `width: '32px'` / `'28px'` icon containers.
- List rows `height < 44px`.

## 11. Empty-state copy per tab

- **Brief:** "Good morning. Let's set up your day — add your first habit to begin." CTA → Habits.
- **Habits:** "Habits are the small daily actions that compound into who you become." Sample chips: Drink water · Read 20 min · Morning walk · No phone in bed.
- **Todos:** "A clean inbox starts here. Capture anything you don't want to forget." Input "What needs doing?"

- **Goals:** "A goal is a future you. Where do you want to be in six months? Twelve?" CTA → SMART form.
